# سيرفر ربط هدايا TikTok بالمؤثرات — دليل الاستخدام

## ما هذا السيرفر؟
سيرفر بسيط يتصل ببث TikTok Live، يستمع لأي هدية تصل، ويشغّل المؤثر المناسب لها تلقائيًا في صفحة Browser Source داخل OBS أو TikTok Studio.

---

## الخطوة 1: رفع الملفات على GitHub
1. أنشئ مستودع (repository) جديد على GitHub، مثلاً باسم `tiktok-overlay-server`
2. ارفع هذه الملفات الثلاثة بنفس الهيكل:
   ```
   tiktok-overlay-server/
   ├── package.json
   ├── server.js
   └── public/
       └── overlay-rose.html
   ```

## الخطوة 2: ربط Render بالمستودع
1. من لوحة Render، اضغط **New +** ثم **Web Service**
2. اختر المستودع الذي رفعته للتو
3. عبّئ الحقول كالتالي:
   - **Name**: أي اسم تختاره (مثلاً `badth-server`)
   - **Region**: الأقرب لك
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: Free (يكفي للتجربة الأولى)
4. اضغط **Create Web Service**

بعد دقيقة أو دقيقتين، يعطيك Render رابطًا مثل:
`https://badth-server.onrender.com`

هذا هو رابط سيرفرك الحقيقي. احتفظ به.

---

## الخطوة 3: تعديل رابط السيرفر داخل صفحة المؤثر
افتح ملف `public/overlay-rose.html`، وغيّر هذا السطر:

```js
const SERVER_URL = "https://badth-server.onrender.com";
```

ضع فيه رابطك الحقيقي من الخطوة 2 بالضبط، ثم أعد رفع الملف على GitHub (Render يحدّث تلقائيًا).

---

## الخطوة 4: تشغيل الاتصال ببث تيكتوك
بعد ما يصير عندك username حقيقي جاهز للبث، تفتح هذا الرابط في المتصفح (أو تستخدم أي أداة لإرسال طلب POST، مثل Postman):

```
POST https://badth-server.onrender.com/api/connect
Body (JSON): { "username": "اسم_المستخدم_على_تيكتوك" }
```

إذا نجح الاتصال، يظهر في سجلات Render (Logs) رسالة:
```
✅ تم الاتصال ببث: اسم_المستخدم | Room ID: ...
```

**ملاحظة مهمة:** الاتصال يعمل فقط إذا كان الستريمر يبث مباشرة (Live) في تلك اللحظة.

---

## الخطوة 5: إضافة صفحة المؤثر إلى OBS / TikTok Studio
1. في OBS، أضف مصدر جديد **Browser Source**
2. ضع الرابط:
   ```
   https://badth-server.onrender.com/overlay-rose.html
   ```
3. اضبط العرض والارتفاع على حجم الشاشة كاملة (مثلاً 1920×1080)
4. فعّل خيار **Shutdown source when not visible** = إيقاف (Off) — لتبقى الصفحة متصلة بالسيرفر دائمًا

**للاختبار السريع بدون انتظار هدية حقيقية:** افتح صفحة المؤثر في المتصفح، واضغط حرف **T** على الكيبورد، سترى بتلات الوردة تتطاير فورًا. هذا يؤكد أن الأنيميشن نفسها تعمل بشكل صحيح، بشكل منفصل عن الاتصال بـ TikTok.

---

## لوحة التحكم (جديد)
بعد نشر السيرفر على Render، رابطك الرئيسي (`https://your-app.onrender.com/`) يفتح **لوحة تحكم حقيقية** فيها قسمين:

1. **الاتصال بالبث** — أدخل username تيكتوك واضغط "اتصال"، بدل استخدام Postman يدويًا
2. **ربط الهدايا** — قائمة بكل الهدايا المعروفة، كل واحدة فيها قائمة منسدلة لاختيار المؤثر المرتبط بها، وزر ▷ لتجربة التشغيل الفعلي الآن على أي صفحة Browser Source مفتوحة حاليًا في OBS

أي تعديل تسويه من هذي اللوحة يُحفظ في ملف `gift-mapping.json` على السيرفر، ويبقى محفوظًا حتى بعد إعادة تشغيل السيرفر.

**ملاحظة:** إضافة مؤثر جديد للوحة (غير الموجودين حاليًا) تحتاج فعليًا تعديل صغير في الكود (مصفوفة `AVAILABLE_OVERLAYS` داخل `server.js`)، لأنه لازم تربط بين اسم المؤثر وملفه الحقيقي.

---

## الربط الحالي الجاهز
المؤثر الحقيقي الجاهز الآن هو **قلب ذهبي متفتت**، مربوط بهدية **وردة** (giftId: 5655):

- ملف المؤثر: `public/overlay-golden-heart.html`
- مرسوم بالكامل بـ SVG/CSS، شفافية حقيقية 100%، لا يحتاج أي فيديو خارجي
- في OBS، Browser Source برابط: `https://your-app.onrender.com/overlay-golden-heart.html`

## ربط هدايا إضافية بمؤثرات أخرى
داخل `server.js`، يوجد جدول بهذا الشكل:

```js
const giftToOverlayMap = {
  5655: "ov1",  // وردة -> قلب بتاج (overlay-heart-rose.html)
  5827: "ov2",  // تاج -> تاج ملكي (overlay-rose.html — اسم قديم، يحتاج تعديل لاحقًا)
};
```

لإضافة هدية جديدة:
1. تحتاج تعرف الـ `giftId` الرسمي لها من TikTok (يظهر في سجلات السيرفر تلقائيًا أول مرة تصل أي هدية، حتى لو غير مربوطة)
2. تضيف سطر جديد بنفس الصيغة: `giftId: "اسم_المؤثر"`
3. تنشئ صفحة HTML مستقلة لذلك المؤثر بنفس طريقة `overlay-rose.html` (نسخ وتعديل التصميم والأنيميشن)

---

## ما الذي لم يُبنَ بعد (الخطوات القادمة)
- قاعدة بيانات حقيقية لحفظ mapping كل ستريمر (الآن الجدول ثابت في الكود لستريمر واحد)
- واجهة الويب لتعديل الربط بدون الحاجة لتعديل الكود مباشرة (المرتبطة بالنموذج التفاعلي الذي بنيناه سابقًا)
- نظام تسجيل دخول للستريمرز المتعددين على نفس المنصة
- اشتراكات USDT الشهرية (الـ tiers الثلاثة)
